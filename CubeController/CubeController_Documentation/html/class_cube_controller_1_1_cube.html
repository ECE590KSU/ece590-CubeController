This is an attempt to characterize Bresenham's Line Algorithm in 3D, extrapolating information from the 2D version. A 3D implementation has been modified from its source: https://www.ict.griffith.edu.au/anthony/info/graphics/bresenham.procs

Basic idea in 2D: Let A = 2 times change in B Let B = A - 2 times change in A Let M = A - change in A

Set the starting point.

Then, for every position along A: while (!atEnd): M < 0 new pixel on same line as last pixel; M += A; M >= 0 new pixel on line higher than last pixel; M += B;
